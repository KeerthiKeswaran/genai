```sql
SELECT 
    c.customer_id, 
    SUM(oi.total_amount) AS total_spending,
    COUNT(DISTINCT oi.item_id) AS unique_products
FROM orders o
JOIN order_items oi ON o.order_id = oi.order_id
JOIN customers c ON o.customer_id = c.customer_id
WHERE o.order_date >= CURRENT_DATE - INTERVAL '30 days' 
GROUP BY c.customer_id, SUM(oi.total_amount)
ORDER BY total_spending DESC
LIMIT 5;

```

**Explanation:**

* **Table Aliases**:  We use `c` as an alias for the `customers` table to improve readability and clarity.
* **Explicit Joins**: We explicitly use `JOIN` clauses with `o`, `oi`, and `c` to clearly define the relationships between tables. 
* **Filter Date (WHERE clause):** The `WHERE o.order_date >= CURRENT_DATE - INTERVAL '30 days'` ensures we only retrieve orders from the last 30 days.

**Indexes Utilized:**

* **FOREIGN KEY Indexes**: We ensure that foreign keys are indexed using `o.customer_id` and `oi.order_id`, since these columns are used in multiple joins (the `JOIN` clause defines them). This improves query performance by enabling faster lookups across the `orders` and `order_items` tables when joining with the `customers` table, which is a key step for this specific operation.
